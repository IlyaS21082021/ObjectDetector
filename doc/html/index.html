testObjDetector
Application detects objects on image in accordance with given polygon location
testObjDetector Documentation

  1. Структура проекта Проект состоит из 4-х папок: client, common, grpc, server. client: содержит файлы для сборки клиентской части приложения и папку с тестами. common: содержит файлы, используемые клиетской и серверной частью приложения. grpc: папка с файлами, полученными на основе обработки файла .proto для реализации grpc махенизма. В этой же папке находится скрипт для генерации файлов. server: содержит файлы для сборки серверной части приложения и папку с тестами.
  2. Сборка Части приложения client и server собираются отдельно, каждая в своей папке.

2.1. Сборка client части. В папке client создать папку build. Из неё запустить файл CMakeLists.txt, затем make. Приложение будет собрано и скопировано в папку client.

2.2. Сборка server части. В папке server создать папку build. Из неё запустить файл CMakeLists.txt, затем make. Приложение будет собрано и скопировано в папку server.

  1. Входные данные Входные данные задаются файлом yaml. Файл должен находиться рядом с исполняемым файлом клиентской части и называться config.yaml. В файле config.yaml указывается файл с изображением и полигоны с параметрами. Пример структуры данных: port: 22000 image: test.jpg polygons:
    • id: p1 vertices:
      • { x: 124, y: 300 }
      • { x: 300, y: 500 }
      • { x: 400, y: 250 }
      • { x: 100, y: 160 } threshold: 0.21 priority: 3 type: include obj_classes: [0,1,2,5,7,8]
    • id: p2 vertices:
      • { x: 600, y: 140 }
      • { x: 550, y: 200 }
      • { x: 700, y: 300 }
      • { x: 850, y: 260 } threshold: 0.3 priority: 2 type: include obj_classes: [0,1,2,4,6]

Блок polygons задаёт массив полигонов. Блок vertices задаёт массив вершин с координатами. Вершин должно быть не меньше 4-х. threshold - задаёт минимальное отношение площади пересечения данного полигона с боксом объекта на изображении к площади этого бокса для отображения бокса объекта. priority - приоритет полигона. type - тип плоигона: include/exclude obj_classes - массив классов объектов с которыми взаимодействует полигон. port - номер порта, на котором находится сервер. Координаты вершин полигона задаются исходя из того, что начало координат находится в левом верхнем углу.

  1. Запуск сервера Сервер запускается с параметром номер порта.
  2. Тесты Тесты для client и server собираются и запускаются отдельно. Для сборки тестов для client, необходимо создать папку build в client/tests и запустить оттуда CMakeLists.txt, находящийся в папке client/tests. Затем make. В результате будет создан файл client_tests. Для сборки тестов для server, необходимо создать папку build в server/tests и запустить оттуда CMakeLists.txt, находящийся в папке server/tests. Затем make. В результате будет создан файл server_tests.
